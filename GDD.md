## §0. 게임 정체성

### 0.1 게임 컨셉
- **재미의 중심:** ② 파괴 쾌감형 (쏘는 재미) — 강한 무기로 적을 쓸어담는 손맛이 핵심 — `카탈로그 §0.1`
- **한 줄 요약:** 장착 무기를 통째로 갈아끼우며 수많은 무기를 시험하는 종스크롤 파괴 쾌감형 슈팅
- **레퍼런스 게임:** 라이덴, 스트라이커즈 1945 계열 (정통 종스크롤 + 화력 중심)
- **차별점:** B-WINGS식 **교체 모듈형 무기** — 파워업을 쌓는 대신 무기를 통째로 교체. 다수의 무기 모델을 갈아끼우며 시험하는 것이 핵심 루프
- **후보(미채택):** ① 물량 탄막형 / ③ 균형형

### 0.2 스크롤 방식
- **채택:** 종스크롤 (위→아래) — `카탈로그 §0.2`
- **후보(미채택):** 횡스크롤 / 탄막 / 전방향 / 하이브리드

### 0.3 기술 사양 _(확정)_
- **엔진/파이프라인:** Unity 6000.3 · URP 2D. 게임플레이 스프라이트는 **Unlit**.
- **좌표계:** Unity 2D 월드 좌표, **Y+ = 위(전진)**. 화면=직교 카메라(ortho size 5).
- **충돌 판정:** **원-원 거리 비교**(중앙 `CollisionManager`, `IDamageable` 인터페이스). 자기탄→대상 / 적탄→기체 / 본체→기체 / AoE.
- **입력:** New Input System(코드 바인딩). 이동 방향키/WASD · 발사 Z/Space · 이탈 X · 봄 C/B · 일시정지 ESC/P · 시작 Z · 재시작 R.

## ⚙️ 제작 기반: 데이터 주도 설계 + 4대 에디터 _(확정 요구사항)_

ASTRA의 모든 콘텐츠(무기·적·보스·스테이지)는 **데이터 카드(ScriptableObject)**로 정의하고, 기획자가 코드 없이 **전용 커스텀 에디터(Unity EditorWindow)**로 직접 저작한다. → 콘텐츠를 언제든 추가·수정 가능한 모듈식 제작 파이프라인.

| 에디터 | 저작 대상(데이터 카드) | 권한 범위 | 대응 § | 상태 |
|--------|----------------------|----------|--------|------|
| 무기 에디터 | `WeaponData` | 무기 특성 + 무기-출력 Lv.1~4 강화 곡선(발사수·위력·연사) | §3.2 | 🟢 데이터 모델 확정 |
| 적기/편대 에디터 | `EnemyData` + `FormationData` | 개체 적 특성(이동 4종) + 스폰 시 편대(무리) 구성 | §5.1 | 🟢 데이터 모델 확정 |
| 보스 에디터 | `BossData` | 파츠 + 페이즈 + 기믹 4종(분리합체·변신·촉수·다관절) | §5.2 | 🟢 데이터 모델 확정 |

## §3. 무기 시스템 — 핵심 메커니즘 확정
- **방식:** B-WINGS식 **교체 모듈형** — 장착 무기를 통째로 교체 (파워업 누적이 아님) — `카탈로그 §3.5 무기 전환`, `§3.2 무기 종류`
- **설계 원칙(핵심 요구):** 무기 1종 = **데이터 카드 1장(ScriptableObject)**. 발사 로직이 카드를 읽어 작동 → **새 무기 추가 = 카드 추가**(기존 동작 재사용 시 코드 무수정, 신규 동작만 작은 스크립트 1개 추가). 언제든 무기를 늘릴 수 있는 모듈식 구조 = ASTRA의 확정 요구사항
- **교체 방식:** ⓐ **즉시 교체형** — 슬롯 1개. 필드에서 무기 모듈을 줍는 즉시 현재 무기와 교체 — `카탈로그 §3.5`
- **픽업 2종:**
  - 🔄 **무기 아이템(W):** 줍는 즉시 해당 무기로 교체
  - ⬆️ **파워업 아이템(P):** 별도 전용 아이템. 현재 장착 무기를 1단계 강화
- **기본 무기:** **정면 연사(Vulcan)** — 위로 곧게 쏘는 표준 사격. 기체는 항상 보유(모듈 미장착·상실 시 발사). 파워업은 활성 무기(기본 무기 포함)에 적용
- **무기-출력(파워 레벨):** **기체의 속성**, **최대 4단계(Lv.1~4)**. 무기를 교체해도 유지(= ⓐ 유지형). 파워업(P) 1개당 +1단계
- **사망 시:** 1라이프 사이클 종료 → 무기-출력 리셋 (※ 전체 리셋 vs 한 단계 down 은 추후 튜닝 — §2 라이프/§7 연동)

### §3.2 무기 에디터 — `WeaponData` 데이터 모델 (확정)
> 기획자가 무기 에디터(EditorWindow)로 저작하는 무기 카드 1장의 구조. 카탈로그 §3.6 기반.

**[공통 특성]** (무기마다 1회 설정)
- `weaponName`, `icon` (전환 UI용)
- `fireMode`: 단발 / 연사 / 차지 / 세미오토
- `shotSpeed`, `baseDamage`, `damageDecay`(거리 감쇠, §4.11)
- `isPiercing`(관통), `isReflecting`(반사), `maxBounces`
- `hitboxSize` (충돌 판정)
- `laserData` / `lockOnData` (해당 무기만, nullable)
- ※ 탄 범위·관통 = **무기 고유 특성(레벨 비례 아님)**

**[무기-출력 강화 곡선]** (Lv.1~4 배열 — 에디터가 레벨별로 조절)
| 축 | 필드 | 설명 |
|----|------|------|
| 동시 발사 수 | `wayCount[1..4]` | 1→…→4 way 등 |
| 위력 | `damageMul[1..4]` | 기본 위력 배율 |
| 연사 속도 | `fireInterval[1..4]` | 발사 간격(작을수록 빠름) |

→ 무기마다 세 축의 4단계 값을 자유 저작 → 개성 있는 성장 곡선 구현. **무기 에디터 데이터 모델 정의 완료.**
- **후보(미채택):** ⓑ 보유 후 전환형 / ⓒ 하이브리드 (교체 방식) · ⓑ리셋형 / ⓒ부분감소형 (교체 시 레벨 처리) · 기본 무기 후보: 좁은 3way / 약한 단발

## §1. 탄환 시스템 _(구현됨)_
- **풀링 필수**(`BulletPool`). 자기탄/적탄 공용(`isPlayerBullet` 구분).
- 속도 직진 + 화면 밖 자동 회수. 옵션: **관통**(piercing) / **벽 반사**(maxBounces) / **유도**(homing, 지정 타깃 가능).
- 무기/적이 색·스프라이트·반지름·데미지를 지정해 발사. 재사용 시 전체 상태 리셋(스프라이트 누락 버그 수정 완료).

## §4. 특수 공격 _(부분 구현)_
- ✅ **봄:** 화면 전체 클리어(반경 내 적 데미지 + 적탄 소거) + 폭발 연출 + 짧은 무적. 스톡 보유, HUD 표시(`C`/`B`).
- ⏳ 실드·슬로우·그레이즈: 후보(미구현).

## §5. 적·AI·보스

### §5.1 적기/편대 에디터 — 데이터 모델 (확정)
> 두 종류의 카드를 저작. 카탈로그 §5 기반.

**🛸 `EnemyData` (개체 적 1종)**
- **공통 특성:** `hp`(내구력) · `score` · `breakable`(파괴 가능/불가, §5.1~5.2)
- **등장 방식:** 화면 밖 등장(§5.3) / 허공 등장 알파 페이드(§5.4)
- **발사 패턴:** 탄환 §1 참조 (조준탄·n-way 등) — 무발사 가능
- **드롭:** 무기(W) / 파워업(P) / 점수 (확률·종류 지정)
- **이동 방식(`moveType`):**

| moveType | 동작 | §참조 |
|----------|------|------|
| `Straight` | 직선/단순 통과 | §5.3 |
| `PathTrajectory` | 사전 경로(노드 배열) 따라 곡예 비행 | §5.7 |
| `Formation` | 편대 군무(FormationData 연결, 선두 추적) | §5.8 |
| `Homing` | 플레이어 유도/추적(선회 제한) | §2.12 |

> 🔸 `FixedCannon`(고정 대포, §5.6): 하이브리드 지형 구간에 합류 — 지형/프랍에 고정되어 배경 속도로 이동하며 조준 사격.

**🐝 `FormationData` (스폰 무리/편대)** — 제안(표준 §5.8)
- `members`: 구성 적(EnemyData) + 수
- `leadPath`: 선두 궤적(경로 데이터)
- `interval`: 적기 간 간격 프레임(링버퍼 지연, §5.8)
- `leaderBreakScatter`: 리더 격파 시 나머지 흩어짐(기본 ON, 튜닝 가능)
- `wipeBonus`: 편대 전멸 시 추가 점수/드롭(기본 ON, 튜닝 가능)

→ 개체 특성 + 무리 구성을 코드 없이 저작. **적기/편대 에디터 데이터 모델 정의 완료.**

### §5.2 보스 에디터 — `BossData` 데이터 모델 (확정)
> 카탈로그 §5.10~5.16 기반. 파츠 + 페이즈 + 기믹.

**기본 구조 (baseline — 항상 지원)**
- **파츠(`BossPart[]`):** 각 부위 = `hp` · `isWeakPoint`(약점) · `isDestructible`(부위 파괴) · `localPos` · `sprite` (코어 + 파괴 가능 부위) — §5.10
- **페이즈(`BossPhase[]`, 상태 머신):** `hpThreshold`(전환 체력) · `patternSet`(탄막 세트) · `move`(이동) — 체력 구간마다 전환 — §5.11
- **페이즈별 탄막:** 탄환 §1 참조 (조준탄·n-way·원형·유도)

**기믹 레퍼토리 (전부 지원, 구현 우선순위 표기)**
| 기믹 | 데이터 | 난이도 | 우선순위 | §참조 |
|------|--------|:--:|:--:|------|
| 분리·합체 | `SeparationData`(분리 부위·합체 좌표) | 보통 | 1차 | §5.12 |
| 변신 | `TransformData`(새 폼·파츠·페이즈 교체) | 보통 | 1차 | §5.13 |
| 촉수 | `TentacleData`(유도 추적, X/Y 독립) | 어려움 | 고급(후순위) | §5.14 |
| 다관절 IK | `IKChainData`(관절 체인·역운동학) | 어려움 | 고급(후순위) | §5.15 |

→ 파츠·페이즈·기믹을 코드 없이 저작. **보스 에디터 데이터 모델 정의 완료.**

## §6. 배경·스테이지

### §6.0 지형 정책 (확정)
- **방식:** ⓒ **하이브리드** — 주로 개활지, 일부 구간만 지형 — `카탈로그 §6.2`
- **구현 기반:** 책 **다중 스크롤**(§7.4) — 깊은 층=우주(충돌 없음, 개활) / 얕은 층=지형(타일맵, 충돌). 구간별 지형층 ON/OFF로 하이브리드 구현
- **후보(미채택):** ⓐ 개활지 전용 / ⓑ 지형 전면

### §6.1 레벨 에디터 — `StageData` 데이터 모델 (확정)
> 배경·프랍 + 적/보스 배치를 구간(섹션) 단위 타임라인으로 저작. 카탈로그 §6 / 책 Ch.7 기반.

**🌌 배경 (다중 스크롤, §7.4)**
- `layers[]`: 배경 레이어 + 레이어별 스크롤 속도(시차) — 깊은 층(우주, 충돌X) / 얕은 층(지형, 충돌O)
- `scrollSpeed`: 기본 강제 스크롤 속도 / 속도 변화·정지 구간

**🧱 구간(`StageSection[]`) — 하이브리드 핵심**
| 필드 | 설명 |
|------|------|
| `type` | `Open`(개활) / `Terrain`(지형) |
| `tilemap` | 지형 구간일 때: 타일셋·충돌 지형·파괴 가능 지형(§6.2, §7.3) |
| `props` | 배경 오브젝트(장식 / 파괴 가능 지상물 — 고정 대포 배치 등) |
| `spawns` | 스폰 타임라인(아래) |

**⏱ 배치 타임라인 (`SpawnEvent[]`)**
- 거리(스크롤 위치)/시간 트리거 → `EnemyData` 단일 또는 `FormationData` 편대 스폰
- 보스/중간보스 등장 트리거(경고 연출 연동)

→ 구간을 깔고, 지형 구간엔 타일·프랍을 칠하고, 타임라인에 적·보스를 배치. **레벨 에디터 데이터 모델 정의 완료 → 4대 에디터 전부 정의됨.**

## §7. 게임 시스템 _(구현됨)_
- **스코어링:** 처치 점수 + **콤보 배율**(연속 처치 → ×1~8, 시간초과/피탄 시 리셋). 편대 전멸 보너스(wipeBonus).
- **익스텐드(1UP):** 점수 임계마다 +1기.
- **아이템:** 무기(W)/파워업(P) 드롭·즉시교체. 적별 드롭 종류·확률 지정.
- **플로우:** **타이틀 → 플레이 → 일시정지 → 게임오버 → 재시작**(상태머신, `GameManager`). **멀티 스테이지** 진행/반복.
- **스폰:** `StageDirector`가 `StageData` 타임라인(웨이브→경고→보스→클리어) 실행.
- ⏳ 난이도 곡선(루프별 강화)은 후속.
